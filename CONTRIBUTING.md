# Contributing to Web AFK Client

First off, thank you for considering contributing to Web AFK Client! It's people like you that make it a great tool for the community.

## Code of Conduct

By participating in this project, you agree to maintain a respectful and welcoming environment for everyone.

## How to Report a Bug

1. **Check Existing Issues**: Ensure the bug hasn't already been reported.
2. **Use the Template**:
   - **Title**: [BUG] Summary of the issue
   - **Steps to Reproduce**: Detailed steps.
   - **Expected vs Actual Behavior**: What should happen vs what did.
   - **Logs**: Attach backend console output if possible.

## Feature Requests

We love new ideas! When suggesting a feature:
- Explain the **use case** (why is it needed?).
- Describe the **proposed behavior**.
- If possible, suggest how it might be implemented.

## Development Workflow

1. **Fork** the repository.
2. **Create a Branch**: `git checkout -b feature/your-feature-name` or `fix/your-fix-name`.
3. **Set Up Locally**: Follow the instructions in the `README.md`.
4. **Commit Changes**: Use descriptive commit messages.
5. **Push & PR**: Open a Pull Request against the `main` (or `develop`) branch.

### Coding Standards
- Use **TypeScript** for all new frontend code.
- Ensure **Prettier/ESLint** passes before submitting.
- Keep components modular and avoid large monolithic files.
